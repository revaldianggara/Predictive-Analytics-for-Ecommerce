Introduction

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Base URL

http://192.168.118.35:3000

Authenticating requests

This API is not authenticated.

Auth

APIs for auth users

Login user

API for logged in a user.

Example request:
curl --request POST \
    "http://192.168.118.35:3000/api/auth/login" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"username\": \"minus\",
    \"password\": \"ratione\"
}"

const url = new URL(
    "http://192.168.118.35:3000/api/auth/login"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "username": "minus",
    "password": "ratione"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (200, Username dan password match):


{
    "message": "login success",
    "access_token": "xxxxx",
    "refresh_token": "yyyyy"
}

Example response (401, Username doesn't exist):


{
    "message": "User tidak ditemukan!"
}

Example response (401, Username dan password not match):


{
    "message": "Username atau password salah!"
}

Example response (500, Internal server error):


{
    "message": "Could not create token.",
    "error": "some thing wrong!"
}

POST api/auth/login

Body Parameters

username  string  

The username of the user.

password  string  

The password of the user.

Response

Response Fields

message  string  

The message of the response API.

access_token  string  

The token that will be used in every request header to authorize the user, the access_token only valid for 15 minutes.

refresh_token  string  

The token that will be used for get new access_token when expired, the refresh_token expired in 1 month.

Refresh Token

Refresh access token when expired.

Example request:
curl --request POST \
    "http://192.168.118.35:3000/api/auth/refresh" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"refresh_token\": \"id\"
}"

const url = new URL(
    "http://192.168.118.35:3000/api/auth/refresh"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "refresh_token": "id"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (200, refresh_token valid):


{
    "message": "Refresh token success!",
    "access_token": "the_token"
}

Example response (401, Access token is invalid `go to login page`):


{
    "message": "Token is Invalid",
    "subcode": 4012
}

Example response (401, Access token is expired `go to login page`):


{
    "message": "Token is Expired",
    "subcode": 4013
}

Example response (401, Access token is not found `go to login page`):


{
    "message": "Authorization Token not found",
    "subcode": 4011
}

Logout user

requires authentication

API for loging out a user.

Example request:
curl --request DELETE \
    "http://192.168.118.35:3000/api/auth/logout" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --header "x-access-token: {access_token from login API}"
const url = new URL(
    "http://192.168.118.35:3000/api/auth/logout"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
    "x-access-token": "{access_token from login API}",
};

fetch(url, {
    method: "DELETE",
    headers,
}).then(response => response.json());

    "message": "User successfully signed out"
